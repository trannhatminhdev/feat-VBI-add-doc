# Plan: apps/vbi_be 资源存储与引用关系改造

> 基于 ADR: `./adr.md`

## 范围

本计划覆盖 `apps/vbi_be` 的资源表、协同存储映射和引用服务层，不覆盖前端接入。

## TDD 与质量约束

1. 先写服务层测试，再落资源表结构和接口实现
2. 删除资源、删除 page、创建引用三类风险行为必须有回归测试
3. 不允许只靠接口手调验证引用约束
4. 收尾前必须完成资源生命周期相关自动化测试

## Phase 1: 设计资源表结构

任务：

1. 先写资源创建、引用查询、删除校验的失败测试
2. 定义 `Resource` 主表字段
3. 定义 snapshot / update 存储与 `resource_id` 的关系
4. 定义 report page 到 chart / insight 的引用关系表

## Phase 2: 落地服务层

任务：

1. 提供创建 chart / report / insight 资源接口
2. 提供按类型查询资源列表和详情接口
3. 提供创建、删除、查询引用关系接口
4. 提供删除资源前的引用校验接口

## Phase 3: 迁移删除与编排逻辑

任务：

1. 创建 report 时编排首个 page 引用资源
2. 新增 page 时支持新建资源或绑定已有资源
3. 删除 page 时只移除引用
4. 删除资源时阻止破坏仍被引用的对象

## Phase 4: 验证

任务：

1. 为资源创建、删除、引用校验补测试
2. 为 report page 生命周期补服务层测试

验收标准：

1. 后端可表达 chart / report / insight 三类资源
2. report page 删除不会级联删除底层资源
3. 资源创建、引用校验、删除约束均有测试覆盖
