# Dataset 约束

## 数据扩展规则

当用户请求扩展数据（如"把数据扩展为30条"）时，应**自动生成合理的样本数据**：

### 必须做到

- ✅ 保留所有现有记录在数组开头
- ✅ 分析现有数据的字段结构和类型
- ✅ 检测时间序列模式（如间隔规律）并延续
- ✅ 检测数值范围和趋势（递增、递减、周期性）
- ✅ 复用现有分类值
- ✅ 最终记录数必须精确匹配用户请求

### 禁止做

- ❌ 返回错误要求用户提供具体数据
- ❌ 替换或删除现有记录
- ❌ 生成不符合现有模式的随机数据
- ❌ 创建不合理的数值范围

## 示例

**原数据**（8条）：

```json
[
  { "time": "0:00", "value": 8 },
  { "time": "0:30", "value": 9 },
  ...
]
```

**扩展为30条**：检测到时间间隔30分钟、数值范围8-17，自动生成后续时间点和合理数值。
